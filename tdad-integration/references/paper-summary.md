# TDAD 论文核心发现

基于论文《TDAD: Test-Driven Agentic Development -- Reducing Code Regressions in AI Coding Agents via Graph-Based Impact Analysis》（ACM AIWare 2026）的核心结论。

## 核心成果

### 70% 回归降低

在 SWE-bench Verified（100 实例，Qwen3-Coder 30B）上，TDAD 将测试级别回归率从 6.08% 降低到 1.82%（562 → 155 P2P 失败）。

关键机制：
- 构建代码-测试依赖图，识别变更影响的测试
- 在代理提交补丁前，告知代理应验证哪些测试
- 代理可自行发现回归并修复，而非事后检测

### TDD 提示悖论（论文消融实验结论）

**关键发现**：添加经典 TDD 流程指令但**不提供**具体测试上下文，反而**增加**了回归（6.08% → 9.94%）。

原因分析：
1. **冗长提示损害小模型**：经典 TDD 流程提示消耗上下文 token，挤占模型需要的仓库上下文
2. **有野心无定位**：仅凭流程指令的代理尝试更激进的修复，触及更多文件，无图定位导致附带损害

**核心教训**：对小模型，**上下文（检查哪些测试）优于流程（如何编排经典 TDD 步骤）**。

### 简化的威力

自动改进循环中发现的最具影响力单次变更：
- 将 SKILL.md 从 107 行详细 9 阶段 TDD 指令简化为 20 行简洁指引（经典 TDD 流程 → TDAD 精简指引）
- 仅此一项就将分辨率从 12% 提升到 50%（4 倍提升）
- 简化内容：fix → grep test_map → verify，而非详细工作流程描述

### 分辨率改进

作为代理技能部署时（25 实例，Qwen3.5-35B-A3B + OpenCode 代理）：
- 分辨率：24% → 32%（+8pp）
- 生成率：40% → 68%（+28pp）
- 回归率：0%（两个配置均为 0%）

改进机制：测试映射不仅减少回归，还帮助代理**更好地理解代码库结构**，从而生成更多有效补丁。

## 对 TDAD 技能设计的启示

### 1. 保持指令简短

给子代理的指令应精简：
- RED 代理：分析需求 → 写测试 → 确认 RED
- GREEN 代理：读失败输出 → 最小实现 → 确认 GREEN
- REFACTOR 代理：优化结构 → 确认测试仍通过

### 2. 上下文重于流程

代理不需要被告知**如何**编排经典 TDD 步骤（写测试 → 红 → 绿 → 重构），只需要被给予**正确的信息**：
- 哪些文件需要测试
- 测试应验证什么行为
- 现有接口是什么

### 3. 子代理隔离消除上下文污染

论文证实：编写测试时的分析思路会渗透到实现阶段。隔离执行确保每个阶段在干净上下文中工作。

### 4. 自动改进循环

TDAD 工具通过自动改进循环（15 次迭代）优化自身配置：
- 4 次变更被接受（27% 接受率）
- 11 次变更被回退
- 关键改进：简化提示、静态测试映射、路径邻近度评分、基于 import 的回退匹配

此模式可用于持续优化 TDAD 技能配置。

# Guide Builder

通过 MCP 文档工具查阅最新官方文档，结合实际源码交叉验证，构建结构完整、内容准确的技术指南文档。

## 何时使用

当用户需要为某个库、框架、工具或技术主题创建结构化的指南文档时使用本技能。适用于从零开始撰写技术文档，也适用于更新或改进已有文档。

## 工作流程

### 第一步：明确文档范围

与用户确认文档主题和范围：

1. **目标主题** — 要撰写哪个库/框架/工具的指南
2. **目标读者** — 面向初学者、中级开发者还是高级用户
3. **重点章节** — 是否有特别需要详述的章节
4. **输出位置** — 文档保存路径

### 第二步：收集文档资料

使用可用的 MCP 文档工具查阅最新官方文档，按以下优先级获取信息：

1. **Context7 MCP** — 查询库的最新文档和代码示例
   - 先调用 `resolve-library-id` 解析库 ID
   - 再调用 `query-docs` 获取具体主题的文档内容
   - 每个主题最多调用 3 次，避免重复查询

2. **zread MCP** — 直接读取 GitHub 仓库的源码和文档
   - 使用 `get_repo_structure` 了解仓库结构
   - 使用 `read_file` 读取关键源码文件
   - 使用 `search_doc` 搜索特定主题

3. **Web Search** — 补充 Context7 和 zread 未覆盖的内容

### 第三步：交叉验证

将收集到的文档内容与实际源码进行交叉验证：

1. **API 签名验证** — 确认函数/方法的参数、返回值与源码一致
2. **配置项验证** — 确认配置字段、默认值与源码定义一致
3. **示例验证** — 确认代码示例中的导入路径、类名、方法名确实存在
4. **版本一致性** — 确认文档引用的 API 在当前版本中可用

验证方法：
- 通过 `Grep` 在源码中搜索函数定义和类型声明
- 通过 `Read` 读取源码中的关键实现文件
- 通过 `Glob` 确认文件路径是否存在

### 第四步：生成指南文档

按照以下标准结构生成 Markdown 指南文档：

```markdown
# [主题名称] 指南

## 概述 (Overview)

简要介绍：是什么、解决什么问题、适用场景。控制在 200 字以内。

## 快速开始 (Getting Started)

### 环境要求
### 安装
### 最小可用示例
### 验证安装

## 核心概念 (Key Concepts)

### 概念 1
### 概念 2
### 概念 N

## 配置参考 (Configuration Reference)

以表格形式列出所有配置项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|

## 示例 (Examples)

### 基础用法
### 进阶用法
### 常见场景

## 常见问题 (FAQ)

收集实际使用中的常见问题和解决方案。

## 参考资源

- 官方文档链接
- GitHub 仓库链接
```

### 第五步：审查与输出

生成文档后执行自检：

1. **准确性** — 所有代码示例是否可运行
2. **完整性** — 核心章节是否齐全
3. **一致性** — 术语、命名风格是否统一
4. **可读性** — 排版、层级是否清晰

将最终文档保存到用户指定的路径。

## 文档质量标准

- 每个代码示例必须经过源码验证，确保可运行
- 配置参考必须与源码中的类型定义一致
- 使用祈使句描述操作步骤
- 中文文档中，专业术语保留英文原文并附中文说明
- 每个章节开头用 1-2 句话概括本章内容

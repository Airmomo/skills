# 交叉验证模式参考

本文档提供常见的交叉验证模式和降级处理策略，供生成指南文档时参考。

## 常见验证模式

### 模式 1：API 签名验证

**目标：** 确认文档中的函数/方法调用与源码定义一致。

**验证步骤：**

1. 在源码中搜索函数定义：定位到函数声明或类型定义
2. 核对参数列表：名称、类型、是否可选、默认值
3. 核对返回值类型：是否与文档描述一致
4. 检查是否已废弃（deprecated）：搜索 `@deprecated` 注释或废弃标记

**示例验证对照：**

```typescript
//* 源码中的定义
export function createRouter(options: RouterOptions): Router

//! 文档中错误写法
createRouter(path: string, options: RouterOptions)

//? 正确写法（与源码一致）
createRouter(options: RouterOptions): Router
```

### 模式 2：配置项验证

**目标：** 确认配置项的名称、类型、默认值与源码定义一致。

**验证步骤：**

1. 查找配置接口定义（如 `interface Config`、`type Options`）
2. 逐一核对每个配置项的字段名、类型标注
3. 查找默认值定义（通常在 `defaults`、`defaultConfig` 或解构赋值中）
4. 检查是否有运行时动态计算的默认值

**常见问题：**
- 文档中的配置项名称使用了驼峰，但源码使用短横线（或反之）
- 默认值在文档中写成字符串，但源码中是枚举类型
- 配置项已在某个版本中重命名，但文档使用了旧名称

### 模式 3：导入路径验证

**目标：** 确认文档中的 import/require 路径与源码文件结构一致。

**验证步骤：**

1. 查看仓库的 `package.json` 中的 `exports` 或 `main`/`module` 字段
2. 确认子模块路径是否在 `exports` 中声明
3. 对比文档中的导入语句与实际的文件路径

**示例：**

```typescript
//* 正确（package.json exports 中声明了子路径）
import { z } from "zod"
import { mongoose } from "mongoose"

//! 错误（未在 exports 中声明的路径）
import { schema } from "zod/lib/schema"  // 内部路径，不应直接引用
```

### 模式 4：示例代码验证

**目标：** 确认代码示例中引用的类名、方法名确实存在且用法正确。

**验证步骤：**

1. 确认示例中每个类名/方法名在源码的导出列表中
2. 确认调用参数数量和顺序与源码定义匹配
3. 确认链式调用中的每个方法确实存在于返回类型上
4. 确认异步方法使用了正确的调用方式（`await` / `.then()`）

## 降级处理策略

### 信息源降级矩阵

| 信息源状态 | 可验证内容 | 不可验证内容 | 文档标注 |
|-----------|----------|------------|---------|
| Context7 + 源码均可访问 | 全部 | 无 | 正常输出 |
| 仅 Context7 可用 | API 文档描述 | 默认值、内部实现 | "基于官方文档编写" |
| 仅源码可访问 | 全部（直接验证） | 使用场景说明 | "基于源码分析编写" |
| 仅 Web Search 可用 | 基础用法 | 精确签名、默认值 | "基于第三方资料编写，未经源码验证" |
| 均不可用 | 无 | 全部 | 通知用户，询问是否继续 |

### 验证失败分级

| 级别 | 说明 | 处理方式 |
|------|------|---------|
| 严重 | API 签名完全错误，代码无法运行 | 必须修正后才能输出该章节 |
| 中等 | 参数名/默认值不准确 | 标注"待验证"并在表格中注明 |
| 轻微 | 描述措辞与源码注释不完全一致 | 可保持，记录差异备查 |

### 非公开项目验证

当目标库没有公开源码时，使用以下替代验证方法：

1. **类型定义文件** — 检查 `.d.ts` 文件或 `types/` 目录
2. **JSDoc/TSDoc 注释** — 从代码注释中提取参数说明
3. **测试文件** — 从测试用例中推断正确的 API 用法
4. **package.json** — 确认入口文件和导出路径
5. **CHANGELOG** — 确认版本差异和 API 变更历史

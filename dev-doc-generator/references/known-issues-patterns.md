# 常见问题提炼模式

## 问题来源

从参考项目中提炼已知问题时，检查以下来源：

### 1. 项目技术文档

搜索 `docs/tech/` 目录下的问题分析文档。常见关键词：
- "问题"、"stability"、"race condition"、"dead code"
- "性能"、"performance"、"infinite"
- "泄漏"、"leak"、"cleanup"

### 2. Git 历史中的修复提交

通过 git log 搜索修复类提交：

```
git log --oneline --grep="fix" --grep="bug" --all-match
```

关注：
- 同一文件多次修复 → 说明该区域稳定性差
- 提交消息中提到 "prevent"、"avoid"、"handle" → 潜在设计缺陷
- 涉及进程管理、状态机的修复 → 高价值问题

### 3. 状态机分析

如果参考项目包含状态管理逻辑，重点检查：
- 是否有并发保护（防重入锁）
- 状态检测的时序问题
- 超时管理的暂停/恢复是否安全
- 资源清理是否完整（进程、连接、回调）

### 4. 性能问题

常见性能问题模式：
- 组件未使用 `React.memo` → 父组件更新时子组件不必要重渲染
- useEffect 依赖数组包含整个对象/数组 → 每次 diff 都触发 effect
- 虚拟列表配置不当 → 大量离屏渲染
- 事件监听器未清理 → 内存泄漏

## 问题记录格式

每个问题用统一格式记录：

| 字段 | 说明 |
|------|------|
| 严重度 | 高 / 中 / 低 |
| 位置 | 参考代码中的具体文件和方法 |
| 现象 | 用户可见的异常行为 |
| 根因 | 代码层面的根本原因 |
| 解决方案 | 新项目中应采用的修复策略 |

## 问题分类

| 分类 | 典型问题 |
|------|----------|
| 状态机 | 并发保护缺失、超时竞态、状态检测时序 |
| 进程管理 | 进程泄漏、清理不完整、信号处理不当 |
| 渲染性能 | 组件重渲染、effect 过度触发、虚拟列表配置 |
| 协议解析 | 死代码、回退逻辑脆弱、缓存不一致 |
| 数据流 | 无限刷新、缓存条件不满足、事件丢失 |
